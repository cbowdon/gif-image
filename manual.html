GIF Decompose

"gif-decompose.rkt" is a module to extract still images from GIF animations. It is a complement to "file/gif", which enables composing GIF animations from bitmap images.

1 Extracting GIF images

(gif? v) → (boolean?)
  v : any/c
Returns #t if v is a path to a valid GIF file or a byte-string containing a valid GIF file.

(gif-images g) → (stream?)
  g : gif?
Returns a finite stream of still images from g. Each element in the stream is a byte-string containing a valid GIF file.

(gif-write-images g name) → (boolean?)
  g : gif?
  name : string?
Like gif-images but writes each element of the stream to a file. Files are named according to name-X.gif where X is the number of the image in the original animation. The return value is #t if all images were written successfully.

2 GIF utilities

Additional helper procedures are provided for extracting metadata and debugging.

(gif-dimensions g) → (pair?)
  g : gif?
Returns a pair containing the x and y dimensions (in pixels) of the image.

(gif-animated? g) → (boolean?)
  g : gif?
Returns #t if g is animated.

(gif-timings g) → (listof rational?)
  g : gif?
Returns a list of delay times (in seconds) for g.

(gif-comments g) → (stream?)
  g : gif?
Returns a stream of the comments in g, where each comment is a byte-string. Encoding the comments is left as an exercise for the reader.

(gif-print-blocks g) → (boolean?)
  g : gif?
Prints the structure of g to current-output-port, in terms of byte number and block type (e.g. header, graphic control, image, comment...). Returns #t after printing the entire GIF file.
